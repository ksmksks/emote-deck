# DECISIONS

## 設計決定理由
EmoteDeck は Twitch Extension の審査要件と運用負荷を重視し、依存を最小化した構成を採用します。

## Node不使用理由
- 実行環境セットアップの簡素化
- 配布物の透明性向上（HTML/CSS/JSのみ）
- CI がなくてもローカルで再現可能
- ZIP 生成を Windows 標準機能で完結できる

## タブUI採用理由
- 1画面に情報を詰め込みすぎない
- モバイル画面での可読性向上
- 「カテゴリ同時表示禁止」要件を明確に満たせる

## CSS変数採用理由
- Config 変更の即時反映が容易
- Panel / Config 間でテーマ仕様を統一できる
- 色・角丸・グローの変更点を集中管理できる

## 縦制御理由
- パネル高さが過大になっても UI 崩れを防げる
- スクロール領域を限定し、操作性を維持できる
- popout のスマホ表示でも安定する
